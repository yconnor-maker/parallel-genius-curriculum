Module 06

The Knowledge Extraction Problem

Surviving AI's Zero-Memory Problem

Track: ResearchDuration: 90 minFormat: Lecture + Workshop

Overview

Core Thesis

AI has zero persistent memory between sessions. Every conversation is temporary. If you don't extract knowledge before the session ends, it's gone forever. Building systems that survive the amnesia is the foundational infrastructure problem of AI-powered operations.

"AI agents have ZERO persistent memory between sessions. The checkpoint/sync systems exist BECAUSE of this limitation. Never assume an agent 'remembers.'"

Key Concepts

What You'll Learn

1. Knowledge Extraction Before Deletion

"When a task completes, extract institutional knowledge → route to permanent home → then delete. Don't let the completed folder become a graveyard."

2. Brain Dump Mode

YC dumps rapid-fire ideas/tasks into a file. The Conductor processes each entry into a proper task, workflow design, or decision point. This captures thinking that would otherwise die in the chat.

3. AGENT_SYNC_STATE

A single file that tells every agent what every other agent has been doing. Updated at session close. This is how agents coordinate without persistent memory.

4. Archived Session Recovery

"If a session is archived without close-down, the file system survives. The loss is narrative context. Reconstruct from: file timestamps, dashboard state, Slack posts, folder contents."
