Module 02

Designing Your AI Operating System

Architecture, Pipelines, and Permissions

Track: Foundation Duration: 90 min Format: Lecture + Workshop

Overview

Core Thesis

AI tools without architecture produce chaos. The difference between someone who "uses AI" and someone who runs an AI-powered operation is the existence of a persistent system design — roles, permissions, file structures, information flow — that constrains and channels AI output into productive work.

Key Concepts

What You'll Learn

1. The Five-Stage Pipeline

Every piece of work moves through exactly five stages: Triage → Task Assignments → In Progress → Waiting On Approval → Completed. This is not project management theory. This is the actual running system.

2. The Permission Model

Three roles, no overlap: Agents do assigned work. The Conductor orchestrates. Yamicia approves. Nothing leaves Waiting On Approval without human sign-off.

3. Folder Structure Is Law

"Only the 5-stage pipeline. NO rogue folders. Agents in other sessions create ad hoc folders that break the system because content becomes invisible."

4. The Boot Sequence

Every session starts the same way: mount the system folder, read the sync state, scan all five stages, sweep for dictations, check Slack. This is the system's immune system against AI's zero-memory problem.

"YC looks at ONE place — the dashboard. Don't create separate docs she has to hunt for."
