Module 03

Writing for Machines

Task Specification as a Teachable Skill

Track: Foundation Duration: 90 min Format: Lecture + Workshop

Overview

Core Thesis

The quality of AI output is entirely determined by the quality of the task specification. Writing a good task spec is a teachable skill that most people skip — and then blame the AI for bad results.

"There is a specific kind of failure that looks like research. It produces data. It generates citations. It has methodology sections and confidence intervals. And it answers the wrong question so precisely that no one notices the wrong question was asked."

Key Concepts

What You'll Learn

1. The Self-Contained Task File

Every task file must be self-contained. The agent should not need to go find other documents. Every task includes: header, instructions, specified outputs, operating rules, results log, and completion gate.

2. The Results Log Is the Permanent Record

"Chat dies. Everything must be extracted. Write it like you're briefing someone who wasn't in the room."

3. Precision Framing

Start from what you know, not what you can measure. Name the mechanism, not the correlation. Ask what the question assumes. Every research question has embedded assumptions.

4. Don't Solve Non-Problems

"Before writing a task, ask: does completing this move the pipeline forward?" Agents that don't know what exists produce work that conflicts with reality.

Workshop Exercise

Rewrite a Vague Request

Take a vague request ("make me a social media plan") and rewrite it as a self-contained task spec using the CONDUCTOR.md template. Include: instructions, specified outputs with exact file names, operating rules, and a completion gate.
