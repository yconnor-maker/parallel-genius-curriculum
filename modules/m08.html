Module 08

Building Your AI Brain

Persistent Knowledge Architecture

Track: Building Your AI System Duration: 90 min Format: Lecture + Workshop

The Argument

Core Thesis

Module 06 taught you why knowledge extraction matters. This module teaches you where the extracted knowledge goes. The AI Brain is a structured, web-accessible knowledge layer that serves as persistent memory for systems that have none.

Without a Brain, your knowledge extraction has no destination. You write close-down memos that go into a folder nobody reads. You create sync states that sit in Google Drive. You extract decisions that die in chat. The Brain is the permanent home — the place where institutional knowledge accumulates instead of scattering.

"We cannot move forward until we have a source of truth in one place. So that we're not going back to Claude." — YC coaching Ayesha on the AI Sync call

The Architecture

What Lives in the AI Brain

System Documents

CONDUCTOR.md, INBOX_RULES.md, Agent Constitution. The operating rules your system runs on. These are the documents every agent reads at session boot. They define how the system behaves.

Project Briefs

One-page summaries of every active project. Current status, key decisions, blockers, next steps. These prevent the 15-minute context-rebuilding tax at the start of every session.

Source Inventories

What exists. Where it lives. What format it's in. Whether it's been processed. This is the master catalog that prevents the most common AI failure: creating something that already exists because nobody knew it was there.

Decision Logs

What was decided, when, and why. The antidote to the "Decision That Vanished" problem from Module 06. If a decision isn't in the log, it doesn't count. When two agents disagree, the decision log is the authority.

Workflow Definitions

How work moves through the system. The pipeline stages from Module 02. The production loop from Module 07. The editorial workflow with its 15 phases and exit criteria. These are living documents that evolve as the system matures.

Knowledge Items

Curated, distilled knowledge on specific topics. Not raw chat logs — processed, structured insights with metadata, references, and artifacts. Each Knowledge Item is a small unit of institutional memory that any agent can access.

The Boot Sequence

Memory Retrieval as System Startup

The Brain is useless if nobody reads it. The Boot Sequence is the mechanism that ensures the Brain gets loaded into every session:

Step Action Purpose
1Mount the system folderEstablish file access
2Read AGENT_SYNC_STATEWhat happened since last session
3Read CONDUCTOR.mdOperating rules and system design
4Scan all pipeline stagesWhat's in progress, what's stuck
5Sweep for new inputsDictations, emails, Slack messages
6Check for pending approvalsWhat's waiting on the human
"The AI Brain is a structured repository: system docs, project briefs, source inventories, workflow definitions, and decision logs. Not a single file — an architecture."

Implementation

How to Actually Build It

The Free Version

A GitHub repository with markdown files, deployed to Netlify or GitHub Pages. Your AI Brain is literally a website. Agents can read URLs. You give them the URL at session start, they read the Brain, they have context. Total cost: $0.

The Database Version

Coda, Airtable, or Notion as the structured data layer. Each knowledge item is a row. Each project brief is a page. The database provides filtering, sorting, and relational links that flat files can't. Cost: free tier is usually sufficient.

The key insight: Start with the free version. A GitHub repo with 5 markdown files is infinitely more valuable than a sophisticated database you never build. You can always upgrade. You can't upgrade from nothing.

Workshop Exercises

Build Your AI Brain

Exercise 1: The 5-File Brain

Create a folder called "AI_BRAIN" (or whatever you want to call it). Create 5 files: (1) SYSTEM.md — your operating rules. (2) PROJECTS.md — active projects with one-sentence status. (3) DECISIONS.md — key decisions and rationale. (4) SYNC_STATE.md — what happened last session. (5) INVENTORY.md — what content/assets you have and where they live.

Exercise 2: The Boot Test

Start a new AI session. Give it your 5-file Brain. Ask it: "Based on these documents, what should we work on today?" If the AI gives you a sensible answer, your Brain is working. If it asks you to re-explain everything, your files are too vague.

Exercise 3: Deploy Your Brain

Put your AI Brain folder on GitHub. Deploy it to Netlify or GitHub Pages. Now it has a URL. Give that URL to every AI session. Your Brain is now web-accessible, version-controlled, and persistent. Congratulations — you've built infrastructure.
