Module 08

Building Your AI Brain

Persistent Knowledge Architecture

Track: ProductionDuration: 90 minFormat: Lecture + Workshop

Overview

Core Thesis

Your AI is only as good as the knowledge system it can access. Building a persistent, web-accessible knowledge layer — an "AI brain" — turns temporary AI sessions into durable institutional memory.

Key Concepts

What You'll Learn

1. The AI Brain Concept

A web-accessible knowledge layer (deployed on Netlify) that serves as persistent memory. Contains: system docs, project briefs, source inventories, decision logs, workflow definitions. The AI reads this at every session boot.

2. The Boot Sequence as Memory Retrieval

Every session starts: mount the system folder → read AGENT_SYNC_STATE → read CONDUCTOR.md → scan all pipeline stages → sweep for dictations → check Slack. Without this, AI starts from zero.

3. Single Source of Truth

"We cannot move forward until we have a source of truth in one place. So that we're not going back to Claude." — from YC coaching Ayesha on the AI Sync call.

4. The Repository Structure

The AI Brain is a structured repository: system docs, project briefs, source inventories, workflow definitions, and decision logs. Not a single file — an architecture.

Build Your Starter Brain

Build a starter "AI brain" for your own work: 3 files. (1) A system doc that describes how you use AI. (2) A project brief for your biggest current project. (3) A "what I've learned" log. Use this as your boot sequence for every future AI session.
