Module 04

Multi-Agent Orchestration

Roles, Rosters, and the Reconciliation Tax

Track: Agent Operations — "How I Scale" Duration: 90 min Format: Lecture + Workshop

The Argument

Core Thesis

One AI is a tool. Multiple AIs are a team — and teams need management. The moment you go from one AI conversation to two, you've introduced the hardest problem in computing: coordination.

Most people discover multi-agent workflows by accident. They start a second chat because the first one got too long. They open Claude in one tab and ChatGPT in another. They create a "project" and then forget it exists. Now they have two agents with different contexts, different histories, and no shared state. They've just created the AI equivalent of two employees who've never met, working on the same project, in different buildings, with no shared drive.

Orchestration is the discipline of making multiple agents work together without producing chaos. It requires roles, assignment rules, synchronization mechanisms, and — most importantly — a human conductor.

"I'm trying to create these personas, for DA and Labora. And for myself. And it's kind of been like a disaster. A little bit. The conductor is supposed to be the persona that orchestrates and tasks distributes to the other agents. And it just can't keep itself straight."

The Framework

The Agent Roster

Every agent in your system must have a defined role, a defined scope, and defined boundaries. Yamicia's roster evolved through trial and error. Here's what it looks like:

The Hidden Cost

The Reconciliation Tax

Here's what nobody tells you about running multiple agents: every agent you add creates coordination overhead. Yamicia calls this the Reconciliation Tax.

Two agents who don't share state will produce conflicting work. Agent A creates a folder structure. Agent B creates a different one. Now you're spending your time reconciling two systems instead of doing the work. This gets worse with every additional agent.

What Goes Wrong

"The conductor has all these logs, but they're not condensing them. A lot of the stuff we talk about is the exact same because it's still fucked up. So we're talking about the same thing because it's still not fixed. So it's just not being consolidated."

Without reconciliation, agents repeat work, contradict each other, and create parallel realities that the human has to manually merge.

The Secretary Pattern

"What I wanna do is have a secretary that essentially takes that structured information and compiles it up and organizes it. The secretary's stuff should just be picked up by the conductor. It should be taking the same updates and condensing them."

The solution isn't fewer agents — it's a reconciliation layer. An agent whose only job is to consolidate what other agents have done.

The Rules

Four Rules for Agent Assignment

1. Scope Lock

Every agent does one thing. The Scientist does research. The Communicator handles outreach. When an agent tries to help with something outside its scope, it will make worse decisions because it lacks the context that the scoped agent has.

2. The Consumption Test

Before adding an agent, ask: "Can I absorb the output?" If you already can't review what two agents produce, adding a third doesn't help — it creates a backlog you'll never clear. The bottleneck is always the human's capacity to consume and approve.

3. Clean Handoffs Only

Agents don't talk to each other. They communicate through artifacts — files, databases, sync states. Agent A writes a file. Agent B reads it. There's no chat between agents. The handoff must be through structured, persistent documents.

4. The Inspector Pattern

"The executive essentially the whole goal is to make sure the conductor is doing its job correctly. It double checks the persona's role, it double checks that I'm getting my dashboard updates." Every system needs a quality check that runs independently of the production agents.

Workshop Exercises

Design Your Agent Team

Exercise 1: Map Your Current Agents

List every AI tool, conversation, or project you currently use. For each one, write: (1) What it does. (2) What context it has. (3) What context it's missing. (4) Where its output goes. If any two agents produce output that overlaps, you have a reconciliation problem.

Exercise 2: Write an Agent Constitution

Pick your most important AI agent. Write a one-page constitution that defines: its name, its role (one sentence), what it can do, what it cannot do, what inputs it receives, what outputs it produces, and what rules it must follow. This becomes the agent's system prompt.

Exercise 3: Design a Sync State

If you have two or more agents, design an AGENT_SYNC_STATE — a single file that tells every agent what every other agent has been doing. Define: what fields it contains, when it gets updated, who updates it, and where it lives. Then test it: start a new session with your AI and give it the sync state. Does it know what happened in the last session?
