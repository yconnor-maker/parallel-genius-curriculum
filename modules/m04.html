Module 04

Multi-Agent Orchestration

Roles, Rosters, and Assignment Rules

Track: Editorial Duration: 90 min Format: Lecture + Workshop

Overview

Core Thesis

A single AI agent is useful. A system of agents working in parallel, each with defined roles and constraints, is transformational — but only if you have an orchestration layer that prevents them from stepping on each other.

"I don't want to make more agents. We need the agents to actually be effective... No automated scheduling until output consumption is proven reliable. Gate is whether output is read, not whether automation is possible."

Key Concepts

What You'll Learn

1. The Agent Roster

Real agents with specific capabilities: Prism (research), Archivist (reports/deploys), Lumen (document creation), Scout (briefings), Inspector (QC), Conductor (orchestration).

2. Assignment Rules

Match the work to the tool. Research → Gemini agents. Coda writes → Claude with MCP. Briefings → Scout. Anything needing YC's voice → Flag for dictation. Never assign to an agent.

3. The Consumption Test

Agent proliferation is the default failure mode. The test of an agent is not capability, but consumption — whether its output is actually read and used by the human.

4. The Inspector Pattern

Automated quality control: Inspector sweeps every 2 hours, verifies outputs against specs, moves passing work forward, blocks failing work. The human only sees work that has passed automated QC.
