The Folder Structure

Your AI system starts with a folder. Not a tool, not a subscription, not a framework — a folder with a clear hierarchy that every agent can navigate.

# YOUR AI OPERATING SYSTEM — Root Folder Structure # Copy this. Adapt it. Deploy it. 📁 AI_SYSTEM/ ├── 📁 _SYSTEM/ # Operating rules — agents read this at boot │ ├── CONDUCTOR.md # Master operating document │ ├── AGENT_SYNC_STATE.md # Cross-agent coordination │ ├── INBOX_RULES.md # How inputs get processed │ └── DECISION_LOG.md # What was decided and why │ ├── 📁 _INBOX/ # Raw inputs land here — dictations, notes, ideas │ ├── 📁 dictations/ # Voice memos, transcriptions │ ├── 📁 brain-dumps/ # Rapid-fire idea captures │ └── 📁 external/ # Emails, messages, forwarded content │ ├── 📁 _PIPELINE/ # Work moves through these stages │ ├── 📁 00_triage/ # Needs classification │ ├── 📁 01_research/ # Being researched │ ├── 📁 02_drafting/ # Being written │ ├── 📁 03_review/ # Waiting for human review │ ├── 📁 04_revision/ # Being revised after feedback │ ├── 📁 05_final/ # Approved, ready to deploy │ └── 📁 06_published/ # Live — deployed or delivered │ ├── 📁 _PROJECTS/ # One folder per active project │ ├── 📁 project-alpha/ │ │ ├── BRIEF.md # What, why, who, when │ │ ├── 📁 assets/ │ │ └── 📁 drafts/ │ └── 📁 project-beta/ │ ├── 📁 _KNOWLEDGE/ # Persistent memory — your AI Brain │ ├── 📁 knowledge-items/ # Curated insights with metadata │ ├── 📁 source-inventory/ # What content exists and where │ └── 📁 templates/ # Reusable document templates │ └── 📁 _ARCHIVE/ # Completed work — extracted, then moved here ├── 📁 2026-Q1/ └── 📁 2026-Q2/

Why This Structure Works

The underscore prefix (_SYSTEM, _INBOX) sorts system folders to the top. The pipeline stages are numbered so work flows in one direction. Projects are self-contained. Knowledge persists. Archives prevent clutter without losing history.

Agent Constitution — Define Each Agent's Boundaries

Every agent needs a one-page constitution. This goes in the system prompt, custom instructions, or project description for that agent.

Agent Constitution Template

Fill this out for every AI agent in your roster.

Role: [One sentence — what this agent does]

Scope: [What domains/projects this agent handles]

Inputs: [What this agent receives — files, dictations, data]

Outputs: [What this agent produces — drafts, reports, code]

Cannot: [What this agent is NOT allowed to do]

Sync Protocol: [How this agent reports what it did — updates SYNC_STATE, writes to log, etc.]

Example: The Conductor

YC's primary orchestration agent.

The Conductor — Constitution

Role: Master orchestrator. Routes tasks, maintains system state, manages agent roster.

Scope: All projects, all agents, system-wide coordination.

Inputs: Dictations, brain dumps, agent sync states, pipeline scans.

Outputs: Task assignments, pipeline updates, decision logs, session summaries.

Cannot: Write final content. Make clinical decisions. Deploy to production.

Format Rules: All output to YC must be HTML or slide deck. Markdown only for agent-to-agent communication.

Quality Gates: Every task assignment must include: objective, constraints, deliverable format, and completion criteria.

Sync Protocol: Update AGENT_SYNC_STATE.md at session close. Write breadcrumbs for next session.

Session Close-Down Checklist

Run this before ending every AI session. Takes 3-5 minutes. Saves hours of reconstruction.

The 7-Point Close-Down

  • What was decided this session? (Log decisions in DECISION_LOG.md)
  • What was created? (List files with paths)
  • What was modified? (Note what changed and why)
  • What's still pending? (Incomplete tasks, waiting on input)
  • What's blocked? (Dependencies, missing information, access issues)
  • What should the next session start with? (Top 3 priorities)
  • Update AGENT_SYNC_STATE.md with all of the above

Why This Matters

Without a close-down, your next session starts from zero. You'll spend 15 minutes re-establishing context. You'll forget what was decided. You'll redo work that's already done. The close-down is the extraction mechanism that turns a temporary conversation into persistent institutional knowledge.
