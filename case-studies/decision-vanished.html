Case Study

The Decision That Vanished

82 HTML files with comment boxes. Zero save functionality. Every decision evaporated the moment the browser tab closed. The agents kept re-asking questions she had already answered.

Yamicia's AI agents delivered work product as HTML dashboards — review decks, signout reports, decision cards. Many included comment boxes where she could type her decisions, feedback, and directions. The dashboards looked professional. The interaction felt natural. Type your decision, move on.

There was one problem: none of them saved.

Every comment box was a standard HTML <textarea> with no backend, no localStorage, no persistence mechanism of any kind. Close the tab, lose the work. Restart the browser, start from scratch. The agents would present the same questions in the next session, and she'd answer them again — sometimes differently, because she couldn't remember what she'd decided before.

Why Nobody Caught It

The UX Looked Right

The dashboards had textareas. Textareas look like input fields. Input fields imply persistence. No user expects that a form on their screen doesn't save — we've been trained by decades of web apps to assume that typing into a box means the data goes somewhere. The failure was invisible until you tried to retrieve your work.

The Agents Didn't Know

The AI agents generating these dashboards didn't have a "save user decisions" step in their workflow. They were optimized for presenting information, not capturing responses. The agents treated the HTML as output — a deliverable — not as an interface that needed to function as an input mechanism.

"The most dangerous UX failure in an AI-human workflow is not a wrong answer — it's a lost answer."

The Fix

The master template was retrofitted with:

  • ✦ localStorage auto-save — debounced at 1.5 seconds, so every keystroke persists without performance impact
  • ✦ JSON export — one-click download of all decisions as structured data
  • ✦ Visual save indicator — users can see that their input has been persisted
  • ✦ Template mandate — all future agent-generated HTML must use the persistent template

The Lesson

Before You Build Sophisticated AI Decision Flows, Verify the Human's Decisions Actually Survive the Session

This is a Course 0 lesson. It's not about prompts or agents or orchestration. It's about the most basic infrastructure question in any AI-human system: when the human makes a decision, where does it go? If the answer is "nowhere," everything downstream is theater.
