Operational Case Study

The Reconciliation Tax

Before you bulk-add 400 records, reconcile what's already there. The tax you pay for skipping this step compounds with every record.

Referenced in: M06, M07 Source: Editorial Pipeline Triage, Apr 2026 Pattern: Reconciliation before bulk operations

What Happened

The Editorial Dashboard contained ~650 articles in an Airtable base. The team wanted to add ~400 more from a content migration. An agent was tasked with bulk-adding the new records.

The agent did what agents do: it added all 400 records. Fast. Clean. No errors.

The problem was that ~180 of the new records were duplicates of existing records — same article, different titles, different metadata formatting. And ~120 of the existing records were stubs — empty rows with only a title, created during a previous planning session that was never cleaned up.

2,800
stub records that had to be manually triaged after the merge

The Timeline of Escalation

Day 1 — The Bulk Add

400 records added in 3 minutes

The agent runs the CSV import. Fast and clean. The Airtable base goes from 650 to 1,050 records. Everyone celebrates.

Day 3 — The First Duplicate Found

"Wait, this article already exists"

A human reviewer finds an article with two records — one from the original base, one from the import. Different titles, same content. They merge manually.

Day 7 — The Scale of the Problem Emerges

180 duplicates confirmed across the base

A dedup query reveals the scope. 180 records need manual review because the titles don't match exactly but the content is the same. No automated fix is safe.

Day 14 — The Stub Discovery

2,800 empty rows found

A deeper audit reveals that hundreds of stubs — rows with only a title — were created across multiple planning sessions. They were never cleaned up. They now pollute every query, filter, and report.

Day 21 — The Full Reconciliation

3 full days of human triage

A human reads every record, one by one, deciding: keep, merge, or delete. The 3-minute bulk add required 3 days of cleanup.

The Fix: Reconcile Before Bulk Operations

1. Audit First, Add Second

Before ANY bulk operation, run a dedup query. Compare the incoming data against existing records by title, slug, or content hash. Surface conflicts before creating them.

2. Stub Hygiene is Non-Negotiable

Empty planning rows must be either filled or deleted within 48 hours. Stubs that persist become permanent pollution. Add a rule to your Agent Constitution: "No stub rows older than 48 hours."

3. Upsert, Don't Insert

Use upsert operations (update if exists, insert if new) instead of blind inserts. Define a key column — typically title or slug — that prevents duplicate creation.

4. Batch Verification Checkpoints

After every 100 records, pause and verify. Show the human a sample of 5 records and ask: "Do these look correct?" This catches drift before it scales.

"The cost of reconciliation increases exponentially with every record you add without reconciling first. A 3-minute bulk add can cost 3 days of cleanup. Always reconcile before you add."
