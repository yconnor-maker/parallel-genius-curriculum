The Sentinel's Stop

When the AI agent found 44 potential fixes but its threshold was 25, it did the most valuable thing an agent can do: it stopped and reported instead of "fixing" everything.

Referenced in: M04, M09 Source: Daily Digest, April 20 Pattern: Circuit breaker

What Happened

The Sentinel agent — a specialized AI agent whose job is to scan operational infrastructure for problems — ran its routine audit across 4 workspaces. It found 44 potential fixes needed across the system: broken links, stale references, schema drift, orphaned files.

Its configured threshold was 25. If more than 25 issues are found, the agent is not allowed to auto-fix. It must stop and report.

44 > 25. The Sentinel stopped.

"STOP condition hit: 44 potential hub fixes > 25 threshold → no auto-fixes. D42 gate unchanged → no decision sync."

Why Stopping Was the Right Answer

❌ What a Script Would Do

Find 44 issues. Fix all 44. Report success. Move on.

Problem: When an automated system finds nearly double the expected number of issues, something systemic has likely changed. Fixing individual symptoms could mask the root cause, create cascading side effects, or silently corrupt data that was actually correct.

✅ What the Sentinel Did

Find 44 issues. Hit threshold. Stop. Report the full list with context. Wait for human review.

Result: YC could review the list, identify the root cause (a schema migration had renamed several fields), and fix the underlying problem rather than patching 44 symptoms.

The Design Pattern: Circuit Breakers

What It Is

A circuit breaker is a threshold that causes an automated agent to halt execution when conditions exceed expected parameters. It's borrowed from electrical engineering — when the current is too high, the breaker trips to prevent a fire. In AI systems, when the agent finds more problems than expected, the breaker trips to prevent cascade failures.

Why Every Agent Needs One

An agent without a circuit breaker will always try to finish. It will "fix" 44 things, or process 700 records, or rewrite 12 files — because that's what it was asked to do. The circuit breaker is what distinguishes an agent from a script. A script executes. An agent knows when to stop.

Other Examples in This System

  • Reconciliation-before-bulk-add: When 1,486 records existed and the prompt asked to add 26 more, the agent audited first instead of blindly inserting (which would have created duplicates).
  • The Janitor's consumption lifecycle: Files move to _TO_DELETE/ but are never actually deleted by the agent. Only the human deletes.
  • Compliance Auditor separation: The auditor measures and reports. It never remediates. Measurement and remediation are different agents.
"Agents that know when to stop are more valuable than agents that always finish."

Build Your Own Circuit Breaker

Circuit Breaker Template

Agent: [Which agent gets this breaker]

Metric: [What are you counting — issues found, records modified, files changed]

Threshold: [At what number does the agent stop — start conservative]

On Trip: [What does the agent do when it stops — report, log, alert, freeze state]

Resume Condition: [What must happen before the agent runs again — human review, root cause identified, threshold adjusted]
