01
The Problem

Without architecture, AI produces chaos.

Most people use AI like a search engine with better grammar. They get useful outputs, but those outputs go nowhere — they pile up in chat histories, downloads folders, and scattered docs that nobody can find.

The difference between someone who "uses AI" and someone who runs an AI-powered operation is one thing: a persistent system design.

04
Lesson #18
"Folder structure is LAW. Only the 5-stage pipeline. NO rogue folders. Agents in other sessions create ad hoc folders that break the system because content becomes invisible." — Conductor Lesson #18

This lesson was learned the hard way. Agents kept creating folders like _Needs_Review and _Ready_To_Publish. Content ended up invisible to the dashboard and the orchestrator. The fix: make the folder structure sacred.

The Boot Sequence

How the System Starts Every Session

AI has zero persistent memory. Every session starts from scratch. The boot sequence IS memory retrieval.

1

Mount system folder

Connect to the persistent file system

2

Read AGENT_SYNC_STATE

What did every other agent do since last time?

3

Read CONDUCTOR.md

Who am I? What are the rules? What's the current state?

4

Scan all 5 stages

What's in each folder? What needs attention?

5

Sweep for dictations

Has YC dictated anything new? This is the ignition switch.

6

Report to human

What changed? What's blocked? What needs decisions?

Key Lesson

One Dashboard to Rule Them All

"YC looks at ONE place. Don't create separate docs she has to hunt for." — Conductor Lesson #9
But Wait — Lesson #21

"The dashboard is NOT YC's dashboard — it's the Conductor's task tracker. YC is a human, not an AI tool. Her dashboard should show: her schedule, her concerns, decisions needed. The Conductor Task Tracker is a sub-view, not the homepage."

Lesson 9 and Lesson 21 are the same team. The system learned that even "one dashboard" needs to be human-facing, not system-facing.

The Close-Down

How the System Ends Every Session

If the boot sequence is memory retrieval, the close-down is memory storage.

1

Update AGENT_SYNC_STATE.md

Timestamp, completed items, status changes

2

Update CONDUCTOR.md

What happened this session? Current state section.

3

Verify file system

No files sitting outside the five-stage system.

Conductor Lesson #28

"If a session is archived without close-down, the file system survives. The loss is narrative context from the chat. Reconstruct from: file timestamps, dashboard state, Slack posts, folder contents."

10
Lesson #25
"Auto-execute processing work. Don't ask. Brain dump processing, dictation sweeps, dashboard updates, task routing — these are standing orders. Do them. Save the human's attention for decisions that actually require judgment." — Conductor Lesson #25
11
Design Principle

The System Protects the Human

  • Agents can't create folders — prevents sprawl
  • Agents can't move files (except claiming) — prevents confusion
  • Nothing goes public without human approval — prevents liability
  • Results Log captures everything — chat can be deleted
  • Boot sequence runs automatically — human doesn't brief the AI
  • Dashboard updates without asking — human always sees current state

"She shouldn't have to chase files, re-explain context, or fix what agents broke. That's your job." — The Permission Model
