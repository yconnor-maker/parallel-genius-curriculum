01
The Problem

The quality of AI output is entirely determined by the quality of the task specification.

Most people blame the AI when they get bad output. In nearly every case, the problem was the instruction. Writing a clear task spec is a teachable skill that almost everyone skips.

02
From "How to Ask the Right Question"
"There is a specific kind of failure that looks like research. It produces data. It generates citations. It has methodology sections and confidence intervals and peer review. And it answers the wrong question so precisely that no one notices the wrong question was asked." — Dr. Yamicia Connor, "How to Ask the Right Question"

AI amplifies this failure mode. A poorly framed question produces a confident, wrong answer at scale.

03
The Template

The Self-Contained Task File

Every task file must be self-contained. The agent should never need to go find other documents to know how to behave.

TASK ID: TASK_25 AGENT: Prism (Gemini) DATE: 2026-03-10 PRIORITY: Critical STATUS: Assigned ───────────────────────────── INSTRUCTIONS: [What to do — no ambiguity] OUTPUTS: [Exact files to create, by name] RULES: [Operating rules baked in] RESULTS LOG: [Permanent record — chat dies] COMPLETION: [Three gates, all must be true]
04
Lesson #6
"Chat dies. Everything must be extracted." — Conductor Lesson #6

The Results Log is the permanent record. The chat history will be deleted when the session ends. Whatever was decided, whatever was learned, whatever was produced — if it's not in the Results Log, it's gone.

"Write it like you're briefing someone who wasn't in the room."

05
Precision Framing

Three Steps to the Right Question

1

Start from what you know, not what you can measure

"Measurement is always available. Mechanisms are not. Every question that begins with data instead of understanding has already narrowed itself."

2

Name the mechanism, not the correlation

"The question 'does this work?' will always return yes or no. The question 'how does this work?' will tell you whether the yes means what you think it means."

3

Ask what the question assumes

"Every research question has embedded assumptions. Most of them are invisible because they were never made explicit."

06
Comparison

Vague Request vs. Task Spec

❌ Vague Request

"Make me a social media plan for Q3. Include Instagram, LinkedIn, and TikTok. Make it professional."

No context. No audience. No voice. No constraints. No way to verify quality. Agent will produce generic output.

vs.

✅ Task Spec

TASK: Q3 Social Strategy
AUDIENCE: OB/GYN physicians, 35–55
VOICE: Clinical authority, warm
PLATFORMS: LinkedIn (primary), IG (secondary)
OUTPUT: strategy_Q3.md, 30_posts.csv
CONSTRAINT: Zero AI language, no clichés
GATE: YC approves tone before full draft

07
The Completion Gate

Three Things Must Be True

No task is complete until all three are true. This prevents the "done but wrong" problem.

1

All specified outputs exist

Every file named in the task spec has been created. Not "similar files" — the exact ones.

2

Results Log is complete

Someone who wasn't in the room can understand what was done, what was decided, and what's still open.

3

No self-reported gaps

If the agent flagged something as incomplete or uncertain, the task is NOT done. Fix it first.

Conductor Lesson #31

"QC means QC, not existence-check. If the Results Log says 'missing data,' that's a REJECT. YC should never see work that the agent itself flagged as incomplete."

08
Lesson #13
"Before writing a task, ask: does completing this move the pipeline forward? TASK_26 was verification busywork. Don't create tasks that solve non-problems." — Conductor Lesson #13
09
Lesson #34

Handoff Docs Must Reference Source Tables

The most common failure mode: the task spec says "check the data" but doesn't say where the data is.

The Fix

Every task spec must list every source document. If the agent needs a Coda table, name the table. If the agent needs a file, give the path. If the agent needs context from a previous task, link it.

"Agents don't know what exists unless the task spec tells them." — Lesson #34

10
From "How to Ask the Right Question"

AI as a Precision Tool

"AI is not good at generating the right question. It cannot see what you have seen. It does not have a body. What AI IS good at: once you have the question, it can help you hold it. It can reflect your framing back to you with enough structure that you can evaluate it." — Dr. Yamicia Connor

This is why the human writes the task spec, not the AI. The framing IS the work.

11
Workshop

Write a Task Spec

⏱ 15 Minutes

Exercise: Transform a Vague Request

1

Start with a vague request

Use one from your actual work. Something you'd normally just type into ChatGPT.

2

Rewrite as a self-contained task spec

Use the template: Task ID, Instructions, Specified Outputs (exact file names), Operating Rules, Completion Gate.

3

Swap with a partner

Can they execute your task spec without any additional questions? If they have to ask you anything, the spec is incomplete.

Module 3 Takeaways

What You Now Know

✓

Self-contained > clever

The agent should never need to go find context. Bake everything in.

✓

Results Log = permanent record

Chat dies. The log is what survives. Write it that way.

✓

Precision framing is the human's job

AI can't generate the right question. That's your contribution.

✓

Three-gate completion

Outputs exist + Results Log complete + no self-reported gaps
