03
Lesson #17
"Markdown is for agents, not humans. Anything YC is supposed to read must be HTML or a proper document format."— Conductor Lesson #17

This curriculum itself follows this rule. Every deliverable — slide decks, worksheets, diagrams — is HTML, not markdown. Markdown is the internal format. Humans get visual formats.

Transcription layer

05
The Dashboard

The Dashboard as Dispatch Center

"The dashboard itself should contain the dispatch prompts needed to spin off new agents, eliminating the need to go anywhere else."— Conductor Lesson #10

The dashboard is not a passive status display. It's an active dispatch center. You can SEE what needs doing and TRIGGER the work from the same screen.

06
Anti-Pattern

The Wall of Text Problem

  • 12-paragraph status update — nobody reads it
  • Markdown README in a Slack message — formatting lost, never read
  • Email with 7 attachments — none opened
  • 50-line results log with no structure — skimmed, details missed

Every one of these represents work that was DONE but never ABSORBED. The work was wasted because the format failed.

"If someone can't absorb your output mid-workday, the output failed — not the person."

07
Workshop

The Format Conversion Exercise

⏱ 15 Minutes

Exercise: Convert a Status Update

1

Take a recent status report you wrote

Ideally 2–3 paragraphs or more

2

Convert to 5 slides

One key point per slide. Maximum 15 words per slide. Use the rest for visuals.

3

Convert to a 90-second audio summary

Record yourself giving the same update verbally in 90 seconds or less

4

Compare

Which format would YOU actually look at during a busy workday?

08
Principle

All Review Deliverables → Ops Hub as HTML

Conductor Lesson #20

Every piece of work that needs YC's review should be accessible from one place (the ops hub) in a visual format (HTML). No more hunting through folders for markdown files.

This principle extends to all stakeholders: your client, your boss, your team. They should access your output from one place, in a format that works for their context.

09
Real Example

This Curriculum Is the Proof

  • The curriculum text is in markdown (AI_WORKFLOW_CURRICULUM_v2.md) — for agents
  • The course syllabus is HTML (course_syllabus.html) — for humans
  • The slide decks are interactive HTML with keyboard navigation — for teaching
  • The worksheets are print-ready HTML — for hands-on work
  • The diagrams are visual HTML — for reference and display

Same content, five different formats, each optimized for its audience and use case.
