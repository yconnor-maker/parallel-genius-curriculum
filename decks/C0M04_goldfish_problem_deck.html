The Goldfish Problem

Your Brilliant Employee Has Amnesia

Dr. Yamicia Connor, MD, PhD, MPH
The Labora Collective
01
The Problem

AI forgets everything between sessions.

You spend 45 minutes teaching the AI your project, your preferences, your constraints. You close the tab. Tomorrow you start from zero.

"When you go to a brand new chat, a brand new tool, you realize how much of yourself is reflected in there. Because it just sounds — it's not helpful at all."— Dr. Yamicia Connor
02
Ask the audience: "Who has had the experience of starting a new chat and being frustrated at how generic the AI suddenly sounds?" Everyone will raise their hand.
Case Study

The Decision That Vanished

Real Failure

Review dashboards across the system had comment boxes — places where the human made decisions. But none of those comment boxes had save functionality. Every time the browser tab closed, all decisions disappeared.

82 HTML files with textareas. 14 still needed decisions. The AI kept re-asking questions that had already been answered.

The most dangerous UX failure in an AI-human workflow is not a wrong answer — it's a lost answer.

03
Framework

The Persistence Stack

Before you build anything sophisticated with AI, build the layer that makes decisions survive:

1

Session Persistence

localStorage, auto-save, debounced writes. The minimum: your decisions survive a page refresh.

2

Project Context Files

A CLAUDE.md, a project constitution, a knowledge base file. Information the AI reads at the start of every session.

3

Structured Memory Systems

Knowledge items, conversation logs, decision registries. Systems that accumulate context over weeks and months.

4

Cross-Agent State

Dashboards, Airtable, signout files. Multiple agents share state through a single source of truth.

04
Key Insight

Persistence First. Everything Else Second.

"Before building sophisticated AI decision flows, verify the human's decisions actually survive the session. The fix wasn't more AI — it was localStorage.setItem()."

The most impactful infrastructure you can build is often the simplest: make sure decisions persist. Then make sure context persists. Then worry about intelligence.

06
Warning

The Compounding Cost of Amnesia

1

Re-explaining Costs Time

Every new session, you spend 10-15 minutes re-establishing context that the previous session already had.

2

Forgotten Decisions Get Re-made

Without persistence, you make the same decision twice. Or worse — you make contradictory decisions because you forgot the first one.

3

Agents Inherit Wrong Assumptions

When Agent B doesn't know what Agent A decided, it makes its own assumptions. Those assumptions cascade through downstream work.

07
Workshop

Build Your First Persistence Layer

⏱ 15 Minutes

Exercise: The Context File

1

Create a project file

For your main project, write a markdown file called PROJECT_CONTEXT.md (or CLAUDE.md)

2

Include: who you are, what you're building, what you've decided

Brand voice, technical stack, key decisions already made, things to never do.

3

Feed it to a fresh AI session

Start a new chat. Paste the file. Ask a project-specific question. Compare to a cold start.

08
Takeaway

The One Thing to Remember

"The first infrastructure you build is not an AI agent. It's a system that remembers what you decided."— Dr. Yamicia Connor
