The Shift

One agent is useful. A system of agents is transformational.

But only if you have an orchestration layer that prevents them from stepping on each other. Without constraints, multiple agents produce multiple versions of chaos.

🎼

File reorganization → Conductor only

👩🏽‍⚕️

Anything needing YC's voice → Flag for dictation

04
Lesson #3
"Bundle tasks by resource. If an agent is already in Coda, give them all the Coda work."— Conductor Lesson #3

Context switching is expensive for AI, just like it is for humans. Each agent has a "warm" context — the files it's already loaded, the tools it's already connected to. Bundle work that shares context.

05
Lesson #2

Think Parallelization

"What can other agents be doing right now that doesn't depend on what we're doing?"— Conductor Lesson #2

This is the fundamental question. While Prism researches, Lumen should be writing. While Scout briefs, Archivist should be deploying. The orchestrator's job is to identify parallel tracks.

Lesson #30

"Dispatch aggressively, explain later. Multiple agents should be running in parallel at all times."

06
Pattern

The Inspector Pattern

Automated quality control. The Inspector is not a content agent — it's a verification system.

1

Sweeps every 2 hours

Scans 3_InProgress/ for completed work

2

Verifies against task spec

Does the output match what was specified? All files present?

3

Checks for self-reported gaps

If the results log says "missing data" — that's a REJECT

4

Routes accordingly

Pass → move to 4_WaitingOnApproval. Fail → flag for Conductor.

The human only sees work that has passed automated QC.

07
Anti-Pattern

What Happens Without Orchestration

  • Duplicate work — two agents write the same report
  • Conflicting outputs — one agent deletes what another created
  • Folder chaos — agents create ad hoc directories that break the pipeline
  • Lost context — agent B doesn't know what agent A decided
  • Human overload — YC has to track, reconcile, and fix everything

Orchestration is the difference between a team and a group of individuals bumping into each other.

08
Coordination

How Agents Coordinate Without Memory

AGENT_SYNC_STATE.md

A single file that tells every agent what every other agent has been doing. Updated at session close. Contains: last active timestamps per agent, completed tasks, in-progress work, blocked items.

This is how agents coordinate without persistent memory. The file IS the shared memory.

Key Insight
"The Conductor is a chief of staff, not an agent. Its job is orchestration — writing tasks, moving files, monitoring progress, making recommendations. It does not generate content. It does not deploy. It coordinates."— Conductor Lesson #12

11
Scaling Decision

When Do You Need Multiple Agents?

  • One agent: Single-domain work, sequential tasks, small scope
  • Two agents: Content + research, or creation + verification
  • Three+ agents: Multiple domains, parallel workstreams, continuous operation
  • Conductor needed: When you have 3+ agents and task routing becomes a job in itself

Don't over-engineer. Start with one agent doing excellent work. Add agents when you have parallel workstreams that a single agent can't serve.
