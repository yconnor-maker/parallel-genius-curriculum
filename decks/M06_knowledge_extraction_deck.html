The Knowledge Extraction Problem

Surviving the Amnesia

Dr. Yamicia Connor, MD, PhD, MPH
The Labora Collective
01
The Fundamental Problem
"AI agents have ZERO persistent memory between sessions. The checkpoint/sync systems exist BECAUSE of this limitation. Never assume an agent 'remembers.'"— Conductor Lesson #5
02
What This Means

Every conversation is temporary.

When you close a chat with Claude, ChatGPT, or Gemini — everything discussed is gone. The decisions made, the context built, the problems solved, the reasoning behind the solutions. Gone.

If you don't extract knowledge before the session ends, it's destroyed.

This is not a bug that will be fixed. This is a fundamental constraint of how large language models currently work. Your system must be designed around it.

03
Lesson #14

Knowledge Extraction Before Deletion

"When a task completes, extract institutional knowledge → route to permanent home → then delete. Don't let the completed folder become a graveyard."— Conductor Lesson #14
1

Extract

What was learned? What was decided? What would you tell the next person?

2

Route

Where does this knowledge live permanently? System doc? Project brief? Lessons learned?

3

Then delete

Only after extraction. Not before.

04
Brain Dump Mode

Capturing Rapid-Fire Thinking

YC dumps ideas, tasks, and half-formed thoughts into a single file. The Conductor processes each entry.

How It Works

Each brain dump entry gets triaged into one of three categories:

  • Task — extractable as a standalone task file
  • Workflow design — defines a repeatable process
  • Decision point — requires YC input before proceeding

This captures thinking that would otherwise die in the chat.

05
Lesson #6

The Results Log Is the Permanent Record

The chat will be deleted when the session ends. The Results Log is the only thing that survives.

📝

What was done

Not "I worked on the project" — specific files created, decisions made, problems solved

📝

What was decided

Not "we discussed options" — which option was chosen and why

📝

What's still open

What the next session needs to address. Be specific.

"Write it like you're briefing someone who wasn't in the room."

06
Coordination File

AGENT_SYNC_STATE.md

A single file that tells every agent what every other agent has been doing.

Contents
  • Last updated timestamp
  • Which agent updated it last
  • Per-agent: last active, current task, completed items
  • Blocked items + what they're waiting on
  • System-level notes

This is how agents coordinate without calling each other. The file IS the shared memory.

07
Lesson #28

When Close-Down Fails

"If a session is archived without close-down, the file system survives. The loss is narrative context from the chat. Reconstruct from: file timestamps, dashboard state, Slack posts, folder contents."— Conductor Lesson #28

This is the insurance policy. Even when the system fails, the persistent file system means you lose narrative context — not actual outputs. Design your system so the files tell the story.

08
Lesson #15

Don't Create Duplicate Files

"Before creating anything, check if it already exists. Task specs must state: creating new OR updating existing."— Conductor Lesson #15

Duplication is how institutional knowledge fragments. If two copies exist, they immediately diverge. One source of truth, always.

09
Workshop

The Extraction Memo

⏱ 5 Minutes

Exercise: Write an Extraction Memo

End your NEXT AI session by writing a 5-sentence extraction memo. Practice right now with a recent session:

1

What you worked on

One sentence. Specific. Not "the project."

2

What you decided

One sentence. Which option, and why.

3

What the key output was

File name, location. Not "an output."

4

What's still open

What the next session needs to pick up.

5

What you'd tell the next version of yourself

The specific context that's currently in your head and about to die.
