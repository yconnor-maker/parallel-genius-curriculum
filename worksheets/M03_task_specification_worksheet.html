Part A

Diagnose a Bad Task

Read this AI request. Identify everything wrong with it.

"Write me a blog post about our new product. Make it engaging and SEO-friendly. Around 1500 words. Include a call to action."
List every missing element that would prevent an agent from producing exactly what you want:

Part B

Write a Self-Contained Task Spec

Take a real request from your work and transform it into a proper task specification.

📋 Task Spec Builder

⏱ 15 min
TASK NAME:
INSTRUCTIONS (what to do — be specific, no ambiguity):
SPECIFIED OUTPUTS (exact file names and formats):
OPERATING RULES (constraints the agent must follow):
COMPLETION GATE (three things that must be true):
SOURCE DOCUMENTS (list every file, table, or resource the agent needs):
Part C

Precision Framing Practice

For each of these questions, apply the three-step precision framing method.

QUESTION: "Why aren't our customers renewing?"

Step 1 — What do I already know (not just what I can measure)?
Step 2 — What's the mechanism, not just the correlation?
Step 3 — What does this question assume?

Part D

Partner Review: The Executability Test

👥 Swap your task spec with a partner

⏱ 10 min

Read your partner's task spec. Try to execute it EXACTLY as written. Track every question you have — every place where you'd need to ask the author what they meant.

Reference

Key Concepts

ConceptDefinition
Self-Contained TaskAgent never needs to find other docs. Instructions, rules, outputs, and context all baked in.
Results LogPermanent record that survives after chat deletion. What was done, decided, and left open.
Precision FramingStart from what you know → name the mechanism → ask what the question assumes.
Completion GateThree tests: outputs exist + log complete + no self-reported gaps.
Source ReferencesEvery file, table, or resource the agent needs must be listed in the spec. Lesson #34.
