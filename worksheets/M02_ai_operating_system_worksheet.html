Part A

Audit: Where Is Your Work Right Now?

Before designing a system, map where your AI output currently lives. Check all that apply:

  • Chat histories (ChatGPT, Claude, Gemini conversations)
  • Downloads folder (PDFs, docs, images generated by AI)
  • Email (responses, drafts, generated content)
  • Scattered notes apps (Apple Notes, Notion, Google Docs)
  • Nowhere — I use the output immediately and don't save it
  • A structured system (describe below)
If you have a system, describe it. If you don't, describe what happens when you try to find something you generated with AI three weeks ago.

Part D

Write Your Boot Sequence

Design the 3–6 step checklist that every AI session should start with. What does the AI need to read or check before doing any work?

Part E

Where Does Your System Leak?

Look at your pipeline design from Part B. Answer honestly:

1. Where does work enter your system that ISN'T captured by Stage 1? (e.g., Slack messages, voice notes, sticky notes, "I'll remember this")
2. Where does work get stuck? Which stage is the bottleneck?
3. Where does work leak OUT of the system? (Work that gets done but never makes it to the final stage — it just sits somewhere half-finished)
Part F

Key Concepts Reference

ConceptDefinition
Five-Stage PipelineTriage → Assigned → In Progress → Approval → Completed. Every piece of work, no exceptions.
Permission ModelThree roles (Human, Orchestrator, Agents) with non-overlapping permissions. Prevents chaos.
Boot SequenceThe 6-step startup that loads memory into a zero-memory system. Run it every session.
Close-Down ProcedureUpdate sync state, update system doc, verify file system. Memory storage before session ends.
Folder Structure Is LawOnly the 5-stage pipeline folders. No rogue folders. Lesson #18.
Dashboard ≠ Human DashboardThe task tracker is a sub-view. The human dashboard shows what the HUMAN needs. Lesson #21.
