Reference

Key Concepts

ConceptDefinition
Agent RosterNamed agents with defined capabilities and constraints. Each has a specific engine and toolset.
Assignment RulesMatch work to the agent whose tools and context best fit. Don't generalize — be specific.
Bundle by ResourceGive agents all work that shares the same context/tools. Minimize context switching.
Parallelization"What can other agents be doing right now that doesn't depend on what we're doing?"
Inspector PatternAutomated QC every 2 hours — verifies outputs against specs before human sees them.
AGENT_SYNC_STATEShared file that tells every agent what every other agent has done. Updated at session close.
